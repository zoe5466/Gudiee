# 假貼文種子腳本使用指南

## 簡介
本指南說明如何使用腳本快速添加假貼文到系統中。

## 前置要求
1. ✅ 開發服務器正在運行 (`npm run dev`)
2. ✅ 數據庫已連接
3. ✅ 系統中已有用戶存在

## 方法一：通過 API 添加假貼文（推薦）

這是最簡單的方法，不需要直接訪問數據庫。

### 步驟

1. **啟動開發服務器**
   ```bash
   npm run dev
   ```

2. **在另一個終端運行種子腳本**
   ```bash
   npx ts-node scripts/seed-posts-api.ts
   ```

3. **查看結果**
   - 成功的貼文會顯示 ✅
   - 失敗的貼文會顯示 ❌ 並顯示錯誤信息
   - 最後會顯示統計摘要

## 方法二：通過 Prisma 直接添加（需要數據庫訪問）

### 前置要求
- 環境變量中有有效的 DATABASE_URL
- 已配置 PostgreSQL 連接

### 步驟
```bash
npx ts-node scripts/seed-posts.ts
```

## 常見問題

### Q: 執行腳本時出現 "Can't reach database server" 錯誤？
A: 確保你有有效的數據庫連接。如果數據庫不可用，使用**方法一（API方法）**。

### Q: 執行 API 方法時出現連接錯誤？
A: 確保開發服務器正在運行：
```bash
npm run dev
```

### Q: 貼文沒有出現在首頁上？
A: 可能原因：
1. 貼文狀態不是 "published"
2. 用戶沒有正確的角色設置
3. 需要刷新頁面

### Q: 如何刪除已添加的假貼文？
A: 進入 `/my-content` 頁面，找到相應的貼文並點擊刪除按鈕。

## 自定義貼文內容

要添加自己的貼文，編輯腳本中的 `mockPosts` 數組：

```typescript
const mockPosts = [
  {
    title: '你的貼文標題',
    content: `<h2>標題</h2><p>內容</p>`,
    category: 'food', // food, culture, nature, city, nightlife, shopping, history, beach
    tags: ['標籤1', '標籤2'],
    location: '地點',
    coverImage: 'https://image-url.jpg',
  },
  // 添加更多貼文...
]
```

## 技術細節

### 使用的 API 端點
- `POST /api/posts` - 創建新貼文

### 貼文字段
- `title` (string) - 必需，貼文標題
- `content` (string) - 必需，HTML 格式的貼文內容
- `category` (string) - 必需，分類
- `tags` (array) - 可選，標籤列表
- `location` (string) - 可選，位置
- `coverImage` (string) - 可選，封面圖片 URL
- `status` (string) - 自動設為 "published"

## 支持的分類

- `food` - 美食 🍜
- `culture` - 文化 🏛️
- `nature` - 自然 🏔️
- `city` - 城市 🏙️
- `nightlife` - 夜生活 🌃
- `shopping` - 購物 🛍️
- `history` - 歷史 🏯
- `beach` - 海岸 🏖️

---

**提示**：每次運行腳本都會添加新的貼文。如果要避免重複，先檢查是否已添加相同內容。
